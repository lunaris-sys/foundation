\begin{abstract}
Desktop operating systems accumulate rich contextual data about user activity,
but this data is trapped in application silos with no shared query layer and no
consistent permission model for cross-application access. This work presents
the design and architecture of a Linux-based desktop operating system built
around a structured, local-first Knowledge Graph as a first-class
infrastructure component.

The Knowledge Graph records system activity as typed nodes and relationships
queryable across application boundaries and time. A two-layer write
architecture separates high-frequency event ingestion (SQLite) from semantic
graph traversal (an embedded property graph database), connected by an
asynchronous promotion pipeline. All components access the graph through a
single privileged daemon using cryptographically scoped capability tokens; no
component holds elevated trust by default.

The architecture covers the full system stack: a forked Wayland compositor, a
Tauri-based application framework with a shared design token system, an event
pipeline driven by eBPF, a provider-agnostic AI layer with explicit MCP
integration, profile isolation via systemd-homed, and a layered security model
with an explicit threat model. Each design decision is documented with its
rationale and trade-offs, including honest treatment of unresolved risks.

The system is currently in the design phase. The contribution of this work is
the architecture itself: a concrete, implementable blueprint that treats the
Knowledge Graph as operating system infrastructure rather than an application
feature.
\end{abstract}
